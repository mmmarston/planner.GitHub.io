# 8-Week Cut Plan — Weekly Planner

Your personal meal plan, shopping list, prep guide, and daily schedule — all in one interactive webpage.

## 🚀 Deploy to GitHub Pages (free hosting, 5 minutes)

### Step 1 — Create a GitHub account

Go to [github.com](https://github.com) and sign up if you don’t have an account.

### Step 2 — Create a new repository

1. Click the **+** icon in the top right → **New repository**
1. Name it anything, e.g. `my-meal-plan`
1. Set it to **Public** (required for free GitHub Pages)
1. Check **“Add a README file”**
1. Click **Create repository**

### Step 3 — Upload the file

1. Inside your new repo, click **Add file** → **Upload files**
1. Drag and drop `index.html` into the upload area
1. Scroll down, click **Commit changes**

### Step 4 — Enable GitHub Pages

1. Click **Settings** (tab at the top of your repo)
1. Scroll down to **Pages** in the left sidebar
1. Under **Source**, select **Deploy from a branch**
1. Set branch to **main**, folder to **/ (root)**
1. Click **Save**

### Step 5 — Get your live URL

Wait about 60 seconds, then refresh the Settings → Pages page.  
Your site will be live at:

```
https://YOUR-USERNAME.github.io/my-meal-plan
```

That’s it. Share that link with anyone — it works on any phone, tablet, or computer, no app needed.

## ✅ Features

- **Meal Plan** — 7-day streamlined plan with macros per meal
- **Shopping List** — tap items to check off as you shop
- **Prep List** — Sunday batch cook guide with timers
- **Daily Schedule** — meals + workouts combined by day, tap to mark done

## 🔒 Privacy

This is a static HTML file. No data is sent anywhere, no account required, no tracking. All checkboxes reset when you close the browser tab — this is by design (no database needed).

## 💡 Making changes

To edit any meal, time, or ingredient: open `index.html` in any text editor (Notepad, TextEdit, VS Code), make your changes, save, then re-upload to GitHub following Step 3 above.
